# Agent Notes

## Commit Messages

- Use the format `[scope] summary`.
- Keep both `scope` and `summary` lowercase.
- Pick a scope that matches the primary area changed, usually a top-level config or directory such as `zsh`, `nvim`, `ghostty`, `shell`, or `zellij`.
- Keep the summary short and action-oriented.
- Do not add a trailing period.

Examples:

- `[shell] add \`take\``
- `[zellij] track config`
- `[ghostty] route option to alt`
